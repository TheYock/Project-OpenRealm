Artifact Description

OpenRealm is a real-time multiplayer browser game built with Node.js, Express, Socket.IO, and MongoDB Atlas, created in early 2025. Players share a live canvas, move freely, and communicate through a chat system with text effects and admin controls.

Justification

I selected OpenRealm because the database drives real application behavior rather than just storing static records. This milestone added two enhancements: a persistent moderation system that saves timed mute and freeze durations to MongoDB so players cannot escape punishment by re-logging, and avatar color customization that is saved per user and restored on every login. Both features required meaningful schema decisions, asynchronous database operations, and server-side input validation.

Course Outcomes

The enhancement primarily addresses Outcome Four through practical database design — choosing the right data representation, handling async reads and writes without blocking gameplay, and building features that persist meaningfully across sessions. Outcome Five is addressed through server-side validation of all incoming data and closing the re-login exploit in the moderation system. Outcome Three is touched on through the timer math required to pause and resume countdowns between sessions.

Reflection

The most valuable lesson came from the moderation timer design. Storing an absolute expiry timestamp seemed obvious at first, but it would allow players to shorten their punishment by logging out. Switching to remaining milliseconds — written back to the database on disconnect and restored on reconnect — required rethinking both the schema and the disconnect handler. That process of recognizing a design flaw and correcting it at the data model level was the most genuine database engineering in this milestone. The avatar feature reinforced the importance of deciding what state needs to be persisted versus what can be computed, and when to broadcast updates to keep all clients consistent.
