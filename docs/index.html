Enhancement One

Software Design & Engineering

1
OpenRealm — Full-Stack Architecture
Node.js · Express · Socket.IO · JWT · bcrypt

Artifact Description

The artifact selected for this enhancement is my OpenRealm project, a real-time multiplayer web application developed using Node.js, Express, and Socket.IO. The application allows multiple users to connect to a shared environment, move their characters in real time, and communicate through a chat system. The original version of this project was created as a prototype to demonstrate client-server communication and real-time event handling.

Justification

I selected this artifact for my ePortfolio because it demonstrates several important full-stack development concepts, including real-time networking, client-server architecture, and interactive front-end design. This project aligns well with my professional interests, especially in areas related to system design and network-based applications.

The original version of OpenRealm relied entirely on in-memory data, meaning that player identities and session data were not persistent. As part of this enhancement, I expanded the application by implementing a user authentication system and integrating MongoDB Atlas for persistent data storage. This included adding user registration and login functionality, allowing users to create accounts and return to the application with consistent identities.

I improved the overall software design by separating responsibilities within the application. The server now handles API routes for authentication separately from real-time Socket.IO communication, while the client distinguishes between spectator and authenticated player states. These changes improve the maintainability and scalability of the application and reflect more professional software engineering practices.

Reflection

Enhancing this artifact helped me better understand the difference between a functional prototype and a more complete, production-style application, which has always been an issue for me. Initially, the project focused on demonstrating real-time communication, but adding authentication and database integration required me to think more carefully about application structure, data flow, and user management.

Overall, this enhancement moved OpenRealm closer to a professionally structured full-stack application and provided valuable experience in software design, system integration, and secure application development.

Enhancement Two

Algorithms & Data Structures

2
OpenRealm — Game Systems & Algorithmic Design
Hash Maps · Sorting · Vector Math · Game Loop

Artifact Description

The artifact selected for this enhancement is OpenRealm, a real-time multiplayer web application built with Node.js, Express, Socket.IO, and MongoDB Atlas. The project was originally created as a prototype to explore client-server communication and real-time interactions between players in a shared environment.

Justification

I selected this artifact because it provides a strong example of how algorithms and data structures directly impact performance and user experience. In a real-time system, inefficient logic quickly becomes noticeable, making it an ideal project for demonstrating improvement.

For this milestone, I enhanced the project by implementing point-and-click movement, improving the player tracking system, adding a sortable player list, and introducing basic server-side bots. The most significant improvement was replacing the original WASD movement with a point-and-click system using vector normalization. By calculating the distance between the player and a target position and normalizing the direction vector, the player now moves at a consistent speed regardless of direction. This prevents diagonal movement from being faster than horizontal or vertical movement and creates smoother, more accurate motion.

The player registry uses a hash map (JavaScript object) keyed by socket ID, allowing for O(1) average-case lookup, insertion, and deletion. This is important because player updates occur frequently, and iterating through an array for each update would not scale well. When displaying the player list, the data is converted into an array using Object.entries() and sorted using a comparator function, which runs in O(n log n) time. This approach balances efficient real-time updates with flexible UI presentation.

These enhancements demonstrate the ability to design and evaluate computing solutions using algorithmic principles. Choosing a hash map over an array, implementing vector normalization for movement, and handling sorting as a derived operation all reflect deliberate design decisions based on performance and scalability.

Reflection

During this process, I learned how small implementation details can significantly affect system behavior. For example, my initial movement logic caused diagonal speed inconsistencies, which led me to better understand vector math and normalization. I also encountered challenges managing bot behavior, particularly ensuring that background processes were properly cleaned up, which reinforced the importance of resource management in real-time systems.

Overall, this milestone helped me move beyond simply writing functional code and toward making intentional design decisions based on efficiency, scalability, and system behavior.

Enhancement Three

Databases

3
OpenRealm — MongoDB Persistence Layer
MongoDB Atlas · Mongoose · Persistent State · Avatar Customization

Artifact Description

The artifact selected for this enhancement is OpenRealm, a real-time multiplayer web application built with Node.js, Express, Socket.IO, and MongoDB Atlas. The project was originally created as a prototype to explore client-server communication and real-time interactions between players in a shared environment.

Justification

I selected this artifact because it provides a strong example of how databases support real-time applications and persistent user data. In a multiplayer environment, maintaining consistent user information across sessions is important for both functionality and user experience, making this project a good fit for demonstrating database improvements.

For this milestone, I enhanced the project by implementing persistent avatar customization. Players can now select a custom avatar color, which is saved to the database and restored whenever they log back in. This required updating the user schema to include avatar data, modifying server-side logic to store updates in MongoDB, and ensuring that the data is retrieved and applied when a player reconnects.

The database operations use MongoDB with Mongoose, allowing structured user data to be stored and accessed efficiently. When a player updates their avatar, the server validates the input and performs an update operation in the database. When the player joins the game, their stored avatar data is retrieved and applied to their in-game representation. This ensures consistency between sessions and prevents reliance on client-side data.

These enhancements demonstrate the ability to design and implement database-driven features within a real-time application. The system supports persistent user data, integrates database operations into live gameplay, and ensures that updates are reflected immediately across connected clients.

Reflection

During this process, I learned how important it is to coordinate between the client, server, and database layers. One challenge was making sure that avatar changes were both saved correctly and updated in real time for other players. This required handling both database updates and live event broadcasting. I also had to ensure proper validation to prevent invalid data from being stored.

Overall, this milestone helped me better understand how databases are used in real-world applications, especially in systems that require both real-time interaction and persistent data storage. It reinforced the importance of designing systems that are both responsive and reliable.
